## ⚠️ Migration checklist (do these to replace racksafetyproducts.com)
This is the highest-priority external work — skipping the redirects will lose existing rankings.

1. **Point the domain at Vercel** and pick a canonical host (the code assumes
   `https://www.racksafetyproducts.com`). Set **`NEXT_PUBLIC_SITE_URL`** in Vercel to the
   exact production origin (must match canonical/sitemap). 301-redirect the non-canonical
   host (non-www → www or vice-versa).
2. **301 redirects from old URLs → new.** The current site uses different paths (e.g.
   `/benefits-of-pallet-rack-flue-spaces/flue-guard/`, `/pallet-rack-decking/...`) than this
   build (`/products/<id>`, `/catalog`). Map every old ranking URL to its closest new page
   via `next.config.mjs` `redirects()` or `vercel.json`. **I can crawl the old site and
   generate this redirect map — just say go.**
3. **Search Console:** verify the production domain, submit `/sitemap.xml`, and watch the
   Coverage + Core Web Vitals reports after cutover. Keep the old property to monitor the
   redirect handoff.
4. **Google Business Profile:** update the website URL if it changes; the new LocalBusiness
   schema (address/phone) should match GBP exactly.

## Remaining recommendations (ranked)
1. **Category landing pages** — the catalog is one filtered page; no pages rank for
   "pallet rack flue products", "warehouse column guards", etc. Build `/flue-products`,
   `/pallet-rack-protection`, … (see the **programmatic-seo** skill). Biggest keyword upside.
2. **Core Web Vitals** — Google Fonts load via a render-blocking CSS `@import` in
   `rsp.css`; switch to `next/font` (self-hosted). Images use plain `<img>` (no
   `next/image` → no lazy/WebP/responsive); adopt `next/image` or add `loading="lazy"` +
   compress. Run PageSpeed Insights for a baseline. (Deferred — needs care on React 18.)
3. **Trust / E-E-A-T** — real Privacy Policy + Terms pages (footer links go nowhere), and
   verify or remove unverified claims ("24-Hour Engineering Response", "Lifetime Component
   Warranty", hero stats).
4. **Content engine** — Resources/blog is placeholder; NFPA 13 / OSHA question content
   builds topical authority and AI-Overview citations (see **ai-seo**).
5. **FAQPage schema** — add once the FAQ answers are real (current answers are placeholder;
   don't mark up invented content).
